O que é Programação Orientada a Objetos

A Programação Orientada a Objetos (POO) é um paradigma de programação que organiza os programas em objetos. Esses objetos representam elementos do mundo real dentro de um sistema computacional.

Em vez de desenvolver programas apenas como uma sequência de comandos, a POO busca modelar entidades reais, organizando-as em estruturas chamadas classes.

Essas entidades possuem três características fundamentais:

🗂️

Estado

Os valores armazenados nos atributos em determinado momento.

⚡

Comportamento

As ações que o objeto pode executar, implementadas como métodos.

🔑

Identidade

Cada objeto possui uma identidade única na memória do computador.

Um Carro possui características (cor, marca, velocidade) e comportamentos (acelerar, frear, ligar). A POO permite modelar isso diretamente no código.

Essa forma de organizar programas torna o código mais organizado, mais fácil de manter e mais próximo da forma como pensamos no mundo real.

A POO é amplamente utilizada em linguagens modernas como Java, C++, C#, Python, Kotlin, entre outras.

Nestas aulas utilizaremos Java para ilustrar todos os conceitos.

Vantagens da POO

A POO traz diversos benefícios para o desenvolvimento de software. Aqui estão algumas das principais vantagens:

1
Organização do código

Os programas ficam estruturados em classes, ficando mais fáceis de entender e navegar.

2
Reutilização

Uma classe pode ser reutilizada várias vezes em diferentes partes do programa.

3
Facilidade de manutenção

Programas organizados em objetos são mais fáceis de modificar e corrigir.

4
Modelagem do mundo real

Permite representar objetos reais diretamente dentro do software.

O que é um Objeto

Um objeto é uma coisa material ou abstrata que pode ser percebida e descrita por meio das suas características, comportamentos e estado.

Ou seja, é uma entidade concreta dentro do programa que possui características e comportamentos.

No mundo real, praticamente tudo pode ser considerado um objeto:

um carro  ·  uma caneta  ·  um celular  ·  uma pessoa  ·  um livro

Quando criamos um programa orientado a objetos, tentamos representar esses elementos como objetos dentro do sistema.

Características de um Objeto

Todo objeto possui três elementos fundamentais. Vamos entender cada um deles usando o exemplo de uma caneta.

1. Estado

O estado representa como o objeto está em determinado momento. Ele é definido pelos valores armazenados em seus atributos.

Exemplo — Objeto: Caneta

cor = azul  |  modelo = BIC  |  ponta = 0.5  |  carga = 80%

Esses valores representam como a caneta está em determinado momento.

2. Comportamento

O comportamento representa as ações que o objeto pode executar. Essas ações são implementadas através dos métodos.

Exemplo — Objeto: Caneta

Comportamentos: escrever()   tampar()   destampar()

Cada comportamento representa uma ação que o objeto pode realizar.

3. Identidade

Mesmo que dois objetos possuam exatamente as mesmas características, eles continuam sendo objetos diferentes. Cada objeto possui uma identidade única na memória do computador.

Imagine duas canetas caneta1 e caneta2. Ambas podem ter a mesma cor, o mesmo modelo e a mesma ponta — mas são objetos diferentes porque ocupam espaços distintos na memória. 

        

           Java
          

            

            

            

          

        

        Caneta caneta1 = new Caneta();
Caneta caneta2 = new Caneta();
// caneta1 e caneta2 são objetos DIFERENTES,
// mesmo que tenham os mesmos atributos.

O que é uma Classe

Uma classe é uma estrutura que define como os objetos serão criados. Ela funciona como um molde ou planta para criar objetos, definindo os atributos e os métodos.

Todos os objetos criados a partir da mesma classe seguindo a sua estrutura.

Relação entre Classe e Objeto

Para facilitar a compreensão, podemos usar algumas comparações do mundo real:

Classe
📐 Planta da casa
Objeto
🏠 Casa construída
Classe
📋 Receita de bolo
Objeto
🎂 Bolo pronto

Uma classe pode gerar múltiplos objetos, assim como uma planta pode gerar várias casas ou uma receita pode produzir vários bolos.

Atributos

Os atributos são as variáveis que representam as características de um objeto. Eles armazenam os dados do objeto e definem o seu estado.

Exemplo — Atributos de uma Caneta:

  • modelo
  • cor
  • ponta
  • carga
  • tampada

Essas informações definem o estado do objeto.

Métodos

Os métodos representam as ações que um objeto pode executar. Eles definem o comportamento do objeto.

Exemplo — Métodos de uma Caneta:

  • escrever
  • tampar
  • destampar

Diagramas de Classe (UML)

Os diagramas de classe são representações gráficas utilizadas para representar estruturas da POO.

Eles usam símbolos para indicar os níveis de acesso (visibilidade) de atributos e métodos, informando quem pode acessar um determinado atributo ou método dentro do programa.

Esses símbolos são muito utilizados em UML (Unified Modeling Language) — uma linguagem visual padrão para modelar sistemas.

Estrutura de um Diagrama

Um diagrama de classe possui três partes:

1
Nome da classe

Identificador da classe, geralmente em negrito e centralizado.

2
Atributos

As características do objeto, com seus tipos e visibilidade.

3
Métodos

As ações do objeto, com seus nomes e visibilidade.

Os Símbolos de Visibilidade

Os principais símbolos utilizados são:

Símbolo Visibilidade Descrição Em Java
+ Público Acessível por qualquer classe do programa public
- Privado Acessível somente dentro da própria classe private
# Protegido Acessível na classe e nas subclasses (herança) protected
~ Pacote Acessível dentro do mesmo pacote (sem modificador)

Entendendo cada símbolo

+
Público  public

O símbolo + indica que o atributo ou método é público. Ele pode ser acessado por qualquer classe do programa.

  • Acessível de qualquer classe
  • Pode ser utilizado em qualquer parte do sistema
  • Geralmente utilizado para métodos que representam ações públicas do objeto 
              

                 Exemplo no diagrama e em Java
                

                  

                  

                  

                

              

              // No diagrama:  + escrever()

// Em Java:
public void escrever() {
    System.out.println("Estou escrevendo");
}
-
Privado  private

O símbolo - indica que o atributo ou método é privado. Ele só pode ser acessado dentro da própria classe.

  • Só pode ser acessado pela própria classe
  • Protege os dados do objeto
  • É o nível de acesso mais comum para atributos 
              

                 Exemplo no diagrama e em Java
                

                  

                  

                  

                

              

              // No diagrama:  - modelo : String

// Em Java:
private String modelo;

Essa prática de deixar atributos privados é chamada de encapsulamento — um dos princípios mais importantes da POO. Você aprenderá mais sobre isso na Unidade 04.

#
Protegido  protected

O símbolo # indica que o elemento é protegido. Ele pode ser acessado pela própria classe ou por classes filhas (subclasses).

  • Visível na própria classe
  • Visível nas classes que herdam dela
  • Usado em cenários de herança
              

                 Exemplo no diagrama e em Java
                

                  

                  

                  

                

              

              // No diagrama:  # carga : int

// Em Java:
protected int carga;
~
Pacote / Default

O símbolo ~ representa o nível de acesso package (default). O elemento pode ser acessado por classes que estão no mesmo pacote.

  • Acessível dentro do mesmo pacote
  • Não é visível fora dele
  • É o nível padrão quando não especificamos nenhum modificador 
              

                 Exemplo no diagrama e em Java
                

                  

                  

                  

                

              

              // No diagrama:  ~ cor : String

// Em Java (sem palavra-chave):
String cor;

Exemplo Completo de Diagrama

Veja o diagrama completo da classe Caneta:

Caneta
- modelo : String
- cor : String
- ponta : float
- tampada : boolean
+ tampar()
+ destampar()
+ escrever()

Interpretação do diagrama:

  • Classe: Caneta
  • Atributos privados (-): modelo, cor, ponta, tampada
  • Métodos públicos (+): tampar(), destampar(), escrever()

Ou seja, os atributos estão privados e os métodos estão públicos — essa é a prática recomendada para encapsulamento.

Exemplo Prático em Java

Na POO os programas são estruturados da seguinte forma:

  • Classes definem os atributos e os métodos.
  • Objetos são instâncias de uma classe.
  • Cada objeto possui estado, comportamento e identidade.

Para este exemplo, vamos modelar uma caneta. Definimos:

Classe: Caneta

Atributos: cor, ponta, carga, tampada

Métodos: tampar, destampar, escrever

Essa classe descreve como uma caneta funciona dentro do sistema.

Definindo a Classe

Em Java, uma classe é definida pela palavra-chave class: 

          

             Caneta.java — Classe com atributos
            

              

              

              

            

          

          class Caneta {

    String  modelo;   // modelo da caneta
    String  cor;      // cor da tinta
    float   ponta;    // espessura da ponta
    int     carga;    // nível de tinta (0-100)
    boolean tampada;  // true se tampada, false se destampada

}

Neste exemplo temos uma classe chamada Caneta. Os elementos declarados dentro da classe são seus atributos.

Atributos da Classe Caneta

Vamos analisar cada atributo utilizado:

Atributo Tipo Descrição Valores possíveis
modelo String Representa o modelo da caneta "BIC", "Faber-Castell"…
cor String Representa a cor da tinta "Azul", "Preta"…
ponta float Espessura da ponta 0.5f, 1.0f…
carga int Nível de tinta da caneta 0 a 100
tampada boolean Indica se a caneta está tampada true / false

Adicionando Métodos

Agora podemos adicionar comportamentos à classe com os métodos:

          

             Caneta.java — com métodos
            

              

              

              

            

          

          class Caneta {

    String  modelo;
    String  cor;
    float   ponta;
    int     carga;
    boolean tampada;

    void tampar() {
        tampada = true;
    }

    void destampar() {
        tampada = false;
    }

    void escrever() {
        if (tampada == true) {
            System.out.println("Erro! Não posso escrever");
        } else {
            System.out.println("Estou escrevendo...");
        }
    }

}

Nesse exemplo temos três métodos:

1
tampar()

Altera o estado da caneta: define tampada = true.

2
destampar()

Remove a tampa da caneta: define tampada = false.

3
escrever()

Verifica se a caneta está tampada. Se estiver, exibe uma mensagem de erro. Caso contrário, a escrita acontece.

Criando Objetos — Construtores

Um construtor é um método especial utilizado para inicializar um objeto no momento de sua criação. Ele é executado automaticamente quando o objeto é criado com o operador new.

Características de um Construtor:

  • Possui o mesmo nome da classe
  • Não possui tipo de retorno (nem void)
  • É executado automaticamente quando o objeto é criado

Criando objetos sem construtor explícito

Depois que a classe é criada e os métodos são definidos, podemos criar objetos usando o operador new: 

          

             Criando objetos
            

              

              

              

            

          

          Caneta c1 = new Caneta();
Caneta c2 = new Caneta();
// Quando isso acontece, o construtor da classe é chamado.

Construtor com parâmetros

Podemos criar um construtor que receba valores e já inicialize os atributos da classe: 

          

             Construtor com parâmetros
            

              

              

              

            

          

          class Caneta {

    String modelo;
    String cor;
    float  ponta;

    // Construtor — mesmo nome da classe, sem tipo de retorno
    Caneta(String m, String c, float p) {
        modelo = m;
        cor    = c;
        ponta  = p;
    }

}

// Criando um objeto usando o construtor:
Caneta c1 = new Caneta("BIC", "Azul", 0.5f);
// Agora c1 já nasce com modelo="BIC", cor="Azul", ponta=0.5

Nos exemplos apresentados podemos visualizar a relação entre as classes e objetos:

Classe → Caneta

Objetos → c1 e c2

Todos os objetos seguem o mesmo modelo que foi pré-definido pela classe.

A Palavra-Chave this

A palavra-chave this é utilizada para referenciar o próprio objeto que está executando o método. Ela é muito utilizada dentro de construtores para evitar ambiguidade entre os parâmetros recebidos e os atributos da classe.

Por que usar o this?

Quando o nome do parâmetro do construtor é igual ao nome do atributo, o Java não consegue distinguir automaticamente qual é qual. O this resolve essa ambiguidade.

❌ Sem this 

              

                
                

                  

                  

                  

                

              

              Caneta(String m, String c) {
    modelo = m;
    cor    = c;
}
// Funciona, mas usa nomes
// diferentes (m, c).

✅ Com this 

              

                
                

                  

                  

                  

                

              

              Caneta(String modelo, String cor) {
    this.modelo = modelo;
    this.cor    = cor;
}
// Mais legível: usa o mesmo
// nome para parâmetro e atributo.

this.modelo refere-se ao atributo da classe.

modelo (sem this) refere-se ao parâmetro recebido pelo construtor.

O this evita essa ambiguidade e torna o código mais legível.

Usando os Objetos

Atribuindo valores aos objetos

Cada objeto pode ter valores diferentes nos seus atributos. Podemos atribuí-los diretamente: 

          

             Atribuindo valores
            

              

              

              

            

          

          c1.modelo = "BIC";
c1.cor    = "Azul";
c1.ponta  = 0.5f;

c2.modelo = "Faber-Castell";
c2.cor    = "Preta";
c2.ponta  = 1.0f;

// Mesmo sendo da mesma classe, os objetos possuem estados diferentes.

Chamando Métodos

Podemos executar métodos de um objeto usando o operador ponto (.): 

          

             Chamando métodos
            

              

              

              

            

          

          c1.destampar();  // primeiro destampamos a caneta
c1.escrever();   // depois chamamos o método escrever

Exemplo Completo

Abaixo está o exemplo completo com a classe Caneta e a classe de teste TesteCaneta, integrando tudo que aprendemos: atributos, métodos, construtor e this. 

          

             Caneta.java — Classe completa
            

              

              

              

            

          

          class Caneta {

    String  modelo;
    String  cor;
    float   ponta;
    boolean tampada;

    Caneta(String modelo, String cor, float ponta) {
        this.modelo  = modelo;
        this.cor     = cor;
        this.ponta   = ponta;
        this.tampada = true;  // caneta começa tampada
    }

    void tampar() {
        tampada = true;
    }

    void destampar() {
        tampada = false;
    }

    void escrever() {
        if (tampada) {
            System.out.println("Erro! A caneta está tampada.");
        } else {
            System.out.println("Estou escrevendo...");
        }
    }

}
        
          

             TesteCaneta.java — Criando e usando objetos
            

              

              

              

            

          

          public class TesteCaneta {

    public static void main(String[] args) {

        // Criando dois objetos da classe Caneta
        Caneta c1 = new Caneta("BIC", "Azul", 0.5f);
        Caneta c2 = new Caneta("Faber", "Preta", 1.0f);

        // Usando os métodos de c1
        c1.destampar();
        c1.escrever();   // Saída: Estou escrevendo...

        // Usando os métodos de c2
        c2.destampar();
        c2.escrever();   // Saída: Estou escrevendo...

    }

}

Saída esperada no console:

Estou escrevendo...
Estou escrevendo...
