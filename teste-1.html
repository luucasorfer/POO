O que é Programação Orientada a Objetos

A Programação Orientada a Objetos (POO) é um paradigma de programação que organiza os programas em objetos. Esses objetos representam elementos do mundo real dentro de um sistema computacional.

Em vez de desenvolver programas apenas como uma sequência de comandos, a POO busca modelar entidades reais, organizando-as em estruturas chamadas classes.

🗂️

Estado

Os valores armazenados em seus atributos em determinado momento.

⚡

Comportamento

As ações que o objeto pode executar, implementadas como métodos.

🔑

Identidade

Cada objeto possui uma identidade única na memória do computador.

Exemplo no mundo real: Um Carro possui características (cor, marca, velocidade) e comportamentos (acelerar, frear, ligar). A POO permite modelar isso diretamente no código.

Vantagens da POO

  • Organização do código: Os programas ficam estruturados em classes.
  • Reutilização: Uma classe pode ser reutilizada várias vezes.
  • Facilidade de manutenção: Programas organizados em objetos são mais fáceis de modificar.
  • Modelagem do mundo real: Permite representar objetos reais dentro do software.

O que é um Objeto

Um objeto é uma entidade concreta dentro do programa que possui características e comportamentos. No mundo real, praticamente tudo pode ser considerado um objeto: um carro, uma caneta, um celular, uma pessoa, um livro.

Características de um Objeto

1. Estado

O estado representa como o objeto está em determinado momento. É definido pelos valores armazenados em seus atributos.

Exemplo — Objeto: Caneta

cor = azul  |  modelo = BIC  |  ponta = 0.5  |  carga = 80%

2. Comportamento

O comportamento representa as ações que o objeto pode executar. Essas ações são implementadas através dos métodos.

Exemplo — Objeto: Caneta

Comportamentos: escrever()   tampar()   destampar()

3. Identidade

Mesmo que dois objetos possuam exatamente as mesmas características, eles continuam sendo objetos diferentes. Cada objeto possui uma identidade única na memória do computador. 

        

           Java
          

            

            

            

          

        

        Caneta caneta1 = new Caneta();
Caneta caneta2 = new Caneta();
// caneta1 e caneta2 são objetos DIFERENTES,
// mesmo que tenham os mesmos atributos.

O que é uma Classe

Uma classe é uma estrutura que define como os objetos serão criados. Ela funciona como um molde ou planta para criar objetos, definindo os atributos e os métodos.

Relação entre Classe e Objeto

Classe
📐 Planta da casa
→
Objeto
🏠 Casa construída
Classe
📋 Receita de bolo
→
Objeto
🎂 Bolo pronto

Uma classe pode gerar múltiplos objetos, assim como uma planta pode gerar várias casas.

Atributos

Os atributos são as variáveis que representam as características de um objeto. Eles armazenam os dados do objeto.

Métodos

Os métodos representam as ações que um objeto pode executar. Eles definem o comportamento do objeto.

Diagramas de Classe (UML)

Os diagramas de classe são representações gráficas utilizadas para descrever estruturas da POO. Em UML (Unified Modeling Language), símbolos especiais indicam o nível de visibilidade de atributos e métodos.

Símbolo Visibilidade Descrição Java
+ Público Acessível por qualquer classe do programa public
- Privado Acessível somente dentro da própria classe private
# Protegido Acessível na classe e nas subclasses (herança) protected
~ Pacote Acessível dentro do mesmo pacote (sem modificador)

Estrutura do Diagrama

Um diagrama de classe possui três partes: nome da classe, atributos e métodos.

Caneta
- modelo : String
- cor : String
- ponta : float
- tampada : boolean
+ tampar()
+ destampar()
+ escrever()

Neste diagrama, os atributos são privados (-) e os métodos são públicos (+). Essa é a prática recomendada para encapsulamento.

Exemplo Prático em Java

Vamos modelar uma Caneta em Java, passo a passo.

Definindo a Classe e seus Atributos

        

           Caneta.java
          

            

            

            

          

        

        class Caneta {

    String modelo;   // modelo da caneta
    String cor;      // cor da tinta
    float  ponta;    // espessura da ponta
    int    carga;    // nível de tinta (0-100)
    boolean tampada; // true se a caneta estiver tampada

}
Atributo Tipo Descrição
modelo String Representa o modelo da caneta
cor String Representa a cor da tinta
ponta float Espessura da ponta (ex: 0.5)
carga int Nível de tinta (0 a 100)
tampada boolean true se tampada, false se destampada

Adicionando Métodos

        

           Caneta.java — com métodos
          

            

            

            

          

        

        class Caneta {

    String modelo;
    String cor;
    float  ponta;
    int    carga;
    boolean tampada;

    void tampar() {
        tampada = true;
    }

    void destampar() {
        tampada = false;
    }

    void escrever() {
        if (tampada == true) {
            System.out.println("Erro! Não posso escrever");
        } else {
            System.out.println("Estou escrevendo...");
        }
    }

}

Criando Objetos — Construtores

Um construtor é um método especial utilizado para inicializar um objeto no momento de sua criação. Ele é executado automaticamente quando o objeto é criado com o operador new.

Características de um Construtor:

  • Possui o mesmo nome da classe
  • Não possui tipo de retorno
  • É executado automaticamente quando o objeto é criado
        

           Construtor com parâmetros
          

            

            

            

          

        

        class Caneta {

    String modelo;
    String cor;
    float ponta;

    // Construtor — mesmo nome da classe, sem tipo de retorno
    Caneta(String m, String c, float p) {
        modelo = m;
        cor    = c;
        ponta  = p;
    }

}

// Criando um objeto usando o construtor:
Caneta c1 = new Caneta("BIC", "Azul", 0.5f);

A Palavra-Chave this

A palavra-chave this é utilizada para referenciar o próprio objeto que está executando o método. É muito utilizada dentro de construtores para evitar ambiguidade entre variáveis locais e atributos da classe.

❌ Sem this 

            

              
              

                

                

                

              

            

            Caneta(String m, String c) {
    modelo = m;
    cor    = c;
}

✅ Com this 

            

              
              

                

                

                

              

            

            Caneta(String modelo, String cor) {
    this.modelo = modelo;
    this.cor    = cor;
}

this.modelo refere-se ao atributo da classe, enquanto modelo refere-se ao parâmetro recebido pelo construtor. O this evita essa ambiguidade.

Exemplo Completo

Abaixo está o exemplo completo com a classe Caneta e a classe de teste TesteCaneta. 

        

           Caneta.java — Classe completa
          

            

            

            

          

        

        class Caneta {

    String  modelo;
    String  cor;
    float   ponta;
    boolean tampada;

    Caneta(String modelo, String cor, float ponta) {
        this.modelo  = modelo;
        this.cor     = cor;
        this.ponta   = ponta;
        this.tampada = true;  // caneta começa tampada
    }

    void tampar() {
        tampada = true;
    }

    void destampar() {
        tampada = false;
    }

    void escrever() {
        if (tampada) {
            System.out.println("Erro! A caneta está tampada.");
        } else {
            System.out.println("Estou escrevendo...");
        }
    }

}
      
        

           TesteCaneta.java — Criando e usando objetos
          

            

            

            

          

        

        public class TesteCaneta {

    public static void main(String[] args) {

        // Criando dois objetos da classe Caneta
        Caneta c1 = new Caneta("BIC", "Azul", 0.5f);
        Caneta c2 = new Caneta("Faber", "Preta", 1.0f);

        // Usando os métodos de c1
        c1.destampar();
        c1.escrever();   // Saída: Estou escrevendo...

        // Usando os métodos de c2
        c2.destampar();
        c2.escrever();   // Saída: Estou escrevendo...

    }

}
