1. Herança

Herança é o mecanismo pelo qual uma classe (subclasse ou classe filha) pode herdar atributos e métodos de outra classe (superclasse ou classe mãe). É o pilar da reutilização de código em POO.

Princípio: se B herda de A, então tudo que A faz, B também faz — e pode fazer mais.

🔁

Reutilização

Escreva o código uma vez na superclasse e todas as subclasses herdam automaticamente.

📐

Especialização

Subclasses podem adicionar comportamentos específicos sem alterar a superclasse.

🏗️

Hierarquia

Organiza classes em árvores que refletem relações do mundo real.

A palavra-chave extends

Em Java, a herança é declarada com extends. Uma subclasse pode ter somente uma superclasse direta (herança simples):

          
 Superclasse Pessoa

          public class Pessoa {

    protected String nome;
    protected int    idade;

    public Pessoa(String nome, int idade) {
        this.nome  = nome;
        this.idade = idade;
    }

    public void apresentar() {
        System.out.println("Nome: " + nome + ", Idade: " + idade);
    }

}
        
          
 Subclasse Aluno — herda de Pessoa

          public class Aluno extends Pessoa {

    private String matricula;
    private double media;

    public Aluno(String nome, int idade, String matricula, double media) {
        super(nome, idade);  // chama o construtor da superclasse
        this.matricula = matricula;
        this.media      = media;
    }

    // método próprio da subclasse
    public void exibirBoletim() {
        apresentar();  // método herdado de Pessoa
        System.out.println("Matrícula: " + matricula);
        System.out.println("Média: "     + media);
    }

}

super — referencia a superclasse. Pode ser usado para chamar o construtor (super(...)) ou um método (super.metodo()) da classe mãe. Quando usado no construtor, deve ser a primeira instrução.

protected — visibilidade para subclasses

O modificador protected torna atributos e métodos acessíveis às subclasses e classes do mesmo pacote, mas não ao mundo externo:

ModificadorPrópria classeSubclasseMesmo pacoteQualquer classe
private✔✘✘✘
protected✔✔✔✘
public✔✔✔✔

Hierarquia de classes

├─Alunomatricula, media, exibirBoletim()
└─Professordepartamento, salario, exibirFicha()

Java suporta apenas herança simples de classes — uma subclasse pode ter apenas uma superclasse direta. Para múltipla herança de comportamento, Java usa interfaces (veremos na seção 5).

2. Sobrescrita de Métodos

Sobrescrita (ou override) ocorre quando uma subclasse redefine um método herdado da superclasse, fornecendo uma implementação específica. O método deve ter o mesmo nome, os mesmos parâmetros e o mesmo tipo de retorno.

Sobrescrita vs Sobrecarga — não confunda:

  • Sobrescrita — mesma assinatura, na subclasse, comportamento diferente
  • Sobrecarga — mesmo nome, parâmetros diferentes, na mesma classe
          
 Sobrescrita com @Override

          public class Animal {
    public void emitirSom() {
        System.out.println("Animal faz um som.");
    }
}

public class Cachorro extends Animal {

    @Override
    public void emitirSom() {          // redefine o comportamento
        System.out.println("Au au!");
    }

}

public class Gato extends Animal {

    @Override
    public void emitirSom() {
        System.out.println("Miau!");
    }

}

@Override é uma anotação que avisa o compilador que você está sobrescrevendo um método da superclasse. Se o método não existir lá (por ex. por um erro de digitação), o compilador acusará erro — é uma proteção valiosa. Sempre use @Override ao sobrescrever.

Chamando o método original com super

A subclasse pode sobrescrever e ainda aproveitar a implementação da superclasse usando super.metodo():

          
 Estendendo o comportamento com super

          public class Aluno extends Pessoa {

    private String matricula;

    @Override
    public void apresentar() {
        super.apresentar();               // executa o método da Pessoa
        System.out.println("Matrícula: " + matricula);  // adiciona mais
    }

}

// Saída de new Aluno("Lucas", 20, "2024001").apresentar():
// Nome: Lucas, Idade: 20
// Matrícula: 2024001

3. Polimorfismo

Polimorfismo significa "muitas formas". Em Java, é a capacidade de uma variável do tipo da superclasse referenciar objetos de subclasses diferentes, e de cada objeto responder à mesma chamada de método de forma diferente. 

          
 Polimorfismo — mesma chamada, comportamentos diferentes

          Animal a1 = new Cachorro();  // variável Animal, objeto Cachorro
Animal a2 = new Gato();      // variável Animal, objeto Gato
Animal a3 = new Animal();    // variável Animal, objeto Animal

a1.emitirSom();  // Au au!
a2.emitirSom();  // Miau!
a3.emitirSom();  // Animal faz um som.

Polimorfismo com listas

O poder do polimorfismo fica evidente quando trabalhamos com coleções de objetos de tipos diferentes:

          
 Lista polimórfica — trata tipos diferentes de forma uniforme

          import java.util.ArrayList;

ArrayList<Animal> animais = new ArrayList<>();

animais.add(new Cachorro());
animais.add(new Gato());
animais.add(new Cachorro());

for (Animal a : animais) {
    a.emitirSom();  // cada objeto chama sua versão do método
}

// Au au!
// Miau!
// Au au!

O código do for não precisa saber se o objeto é Cachorro ou Gato. Ele só sabe que é um Animal e que todo Animal tem emitirSom(). Isso é a essência do polimorfismo.

instanceof e casting

Quando precisar acessar métodos específicos de uma subclasse a partir de uma variável do tipo da superclasse, use instanceof para verificar e depois faça o casting:

          
 instanceof e casting seguro

          Animal animal = new Cachorro();

if (animal instanceof Cachorro) {
    Cachorro c = (Cachorro) animal;  // cast explícito
    c.buscarBola();  // método exclusivo de Cachorro
}

// Java 16+: pattern matching simplifica:
if (animal instanceof Cachorro c) {
    c.buscarBola();  // c já é do tipo Cachorro
}

4. Classes Abstratas

Uma classe abstrata é uma classe que não pode ser instanciada diretamente — ela existe para ser herdada. Serve como modelo obrigatório: define a estrutura comum e pode incluir métodos abstratos que cada subclasse deve implementar.

Método abstrato é um método sem implementação — apenas a assinatura. A subclasse concreta é obrigada a sobrescrevê-lo. Se não fizer isso, a subclasse também deve ser abstrata.

          
 Classe abstrata Forma

          public abstract class Forma {

    protected String cor;

    public Forma(String cor) {
        this.cor = cor;
    }

    // método abstrato — sem implementação, subclasse DEVE implementar
    public abstract double calcularArea();

    // método concreto — herdado normalmente
    public void exibir() {
        System.out.println("Cor: " + cor
            + " | Área: " + calcularArea());
    }

}
        
          
 Subclasses concretas — implementam calcularArea()

          public class Circulo extends Forma {
    private double raio;

    public Circulo(String cor, double raio) {
        super(cor); this.raio = raio;
    }

    @Override
    public double calcularArea() {
        return Math.PI * raio * raio;
    }
}

public class Retangulo extends Forma {
    private double largura, altura;

    public Retangulo(String cor, double largura, double altura) {
        super(cor); this.largura = largura; this.altura = altura;
    }

    @Override
    public double calcularArea() {
        return largura * altura;
    }
}

// Uso polimórfico:
Forma[] formas = {
    new Circulo("Azul", 5),
    new Retangulo("Vermelho", 4, 6)
};

for (Forma f : formas) f.exibir();
// Cor: Azul | Área: 78.53...
// Cor: Vermelho | Área: 24.0

5. Interfaces

Uma interface é um contrato que define um conjunto de métodos que uma classe deve implementar. Diferente de classes abstratas, uma interface não pode ter estado (atributos de instância), mas define quais comportamentos são garantidos. Em Java, uma classe pode implementar múltiplas interfaces.

Analogia: uma interface é como um contrato de emprego. Ao assinar (implements), a classe se compromete a fornecer todos os comportamentos exigidos. O contratante (código cliente) sabe o que pode esperar, sem precisar saber como está implementado.

Criando e implementando interfaces

          
 Definindo interfaces

          // Interface Desenhavel — define um contrato
public interface Desenhavel {
    void desenhar();              // implicitamente public abstract
}

// Interface Redimensionavel
public interface Redimensionavel {
    void redimensionar(double fator);
}

// Classe que implementa DUAS interfaces (múltipla herança de comportamento)
public class Quadrado extends Forma
        implements Desenhavel, Redimensionavel {

    private double lado;

    public Quadrado(String cor, double lado) {
        super(cor); this.lado = lado;
    }

    @Override public double calcularArea() { return lado * lado; }

    @Override public void desenhar() {
        System.out.println("Desenhando quadrado de lado " + lado);
    }

    @Override public void redimensionar(double fator) {
        lado *= fator;
    }

}

Métodos default e static em interfaces

Desde o Java 8, interfaces podem ter métodos default (com implementação padrão) e métodos static:

          
 default e static em interfaces (Java 8+)

          public interface Veiculo {

    void acelerar();       // obrigatório
    void frear();          // obrigatório

    // método default — implementação padrão (pode ser sobrescrita)
    default void ligar() {
        System.out.println("Veículo ligado.");
    }

    // método static — chamado diretamente pela interface
    static void regulamentos() {
        System.out.println("Seguir o Código de Trânsito.");
    }

}

// Chamando o método static pela interface:
Veiculo.regulamentos();

6. Acoplamento Dinâmico

O acoplamento dinâmico (ou late binding) é o mecanismo pelo qual o Java decide em tempo de execução qual versão de um método chamar, com base no tipo real do objeto e não no tipo da variável.

Tempo de compilação vs tempo de execução:

  • O compilador verifica se o método existe no tipo declarado da variável.
  • Em tempo de execução, a JVM chama o método do tipo real do objeto — mesmo que a variável seja do tipo da superclasse.
          
 Late binding — a JVM decide em tempo de execução

          Forma f;

// 1. A variável é do tipo Forma (superclasse)
f = new Circulo("Azul", 5);
f.calcularArea();  // JVM chama Circulo.calcularArea()  → 78.53

f = new Retangulo("Vermelho", 4, 6);
f.calcularArea();  // JVM chama Retangulo.calcularArea() → 24.0

// O compilador só verifica se Forma tem calcularArea()
// A JVM resolve QUAL calcularArea() chamar em tempo de execução

Padrão Factory — acoplamento dinâmico na prática

O acoplamento dinâmico permite criar sistemas onde o código principal não precisa conhecer as subclasses concretas — apenas o tipo base:

          
 Criando objetos por tipo — Factory simples

          static Forma criarForma(String tipo) {
    return switch (tipo) {
        case "circulo"    -> new Circulo("Azul",     5);
        case "retangulo"  -> new Retangulo("Vermelho", 4, 6);
        default           -> throw new IllegalArgumentException("Tipo desconhecido");
    };
}

// O chamador só conhece Forma — não precisa saber qual subclasse
Forma f1 = criarForma("circulo");
Forma f2 = criarForma("retangulo");

f1.exibir();  // Cor: Azul | Área: 78.53...
f2.exibir();  // Cor: Vermelho | Área: 24.0

7. Boas Práticas

Sempre use @Override ao sobrescrever A anotação protege contra erros de digitação e documenta a intenção — se o método não existir na superclasse, o compilador avisa.
Prefira programar para interfaces, não implementações Declare variáveis com o tipo da interface: List<String> lista = new ArrayList<>() — não ArrayList<String>. Isso facilita trocar a implementação sem alterar o código.
Use abstract quando a superclasse não faz sentido sozinha Se não faz sentido criar um new Animal() ou um new Forma(), torne a classe abstrata. Isso impede instanciação acidental e documenta a intenção.
Evite hierarquias muito profundas Heranças com mais de 3 ou 4 níveis tornam o código difícil de entender e manter. Prefira composição quando a hierarquia ficar muito longa.
Interfaces pequenas e focadas (Segregação) Prefira várias interfaces pequenas a uma grande. Uma classe não deve ser obrigada a implementar métodos que não usa — princípio ISP do SOLID.

Hierarquia de Veículos
Nível 1

Crie a superclasse Veiculo com atributos marca e velocidadeMaxima e um método exibirInfo(). Crie as subclasses Carro (adiciona numPortas) e Moto (adiciona cilindradas). Sobrescreva exibirInfo() em cada subclasse para incluir seus atributos extras usando super.

                
 Solução

                public class Veiculo {
    protected String marca;
    protected int    velocidadeMaxima;

    public Veiculo(String marca, int vel) {
        this.marca            = marca;
        this.velocidadeMaxima = vel;
    }

    public void exibirInfo() {
        System.out.println("Marca: " + marca
            + " | Vel. máx: " + velocidadeMaxima + " km/h");
    }
}

public class Carro extends Veiculo {
    private int numPortas;

    public Carro(String marca, int vel, int portas) {
        super(marca, vel);
        this.numPortas = portas;
    }

    @Override public void exibirInfo() {
        super.exibirInfo();
        System.out.println("Portas: " + numPortas);
    }
}

public class Moto extends Veiculo {
    private int cilindradas;

    public Moto(String marca, int vel, int cc) {
        super(marca, vel);
        this.cilindradas = cc;
    }

    @Override public void exibirInfo() {
        super.exibirInfo();
        System.out.println("Cilindradas: " + cilindradas + "cc");
    }
}

// Teste polimórfico
Veiculo[] frota = {
    new Carro("Toyota", 180, 4),
    new Moto("Honda",  210, 600)
};
for (Veiculo v : frota) { v.exibirInfo(); System.out.println("---"); }

Classe Abstrata — Sistema de Pagamento
Nível 1

Crie a classe abstrata Pagamento com o atributo valor e o método abstrato processar(). Implemente três subclasses: PagamentoPix, PagamentoCartao (adiciona parcelas) e PagamentoBoleto (adiciona diasVencimento). Cada uma deve sobrescrever processar() com uma mensagem específica.

                
 Solução

                public abstract class Pagamento {
    protected double valor;

    public Pagamento(double valor) { this.valor = valor; }
    public abstract void processar();
}

class PagamentoPix extends Pagamento {
    public PagamentoPix(double valor) { super(valor); }
    @Override public void processar() {
        System.out.println("Pix de R$ " + valor + " processado instantaneamente.");
    }
}

class PagamentoCartao extends Pagamento {
    private int parcelas;
    public PagamentoCartao(double valor, int parcelas) {
        super(valor); this.parcelas = parcelas;
    }
    @Override public void processar() {
        System.out.println("Cartão: R$ " + valor
            + " em " + parcelas + "x de R$ "
            + String.format("%.2f", valor / parcelas));
    }
}

class PagamentoBoleto extends Pagamento {
    private int diasVencimento;
    public PagamentoBoleto(double valor, int dias) {
        super(valor); this.diasVencimento = dias;
    }
    @Override public void processar() {
        System.out.println("Boleto de R$ " + valor
            + " com vencimento em " + diasVencimento + " dias.");
    }
}

// Teste polimórfico
Pagamento[] pagamentos = {
    new PagamentoPix(150.00),
    new PagamentoCartao(600.00, 3),
    new PagamentoBoleto(200.00, 5)
};
for (Pagamento p : pagamentos) p.processar();
              
Pix de R$ 150.0 processado instantaneamente.
Cartão: R$ 600.0 em 3x de R$ 200.00
Boleto de R$ 200.0 com vencimento em 5 dias.
03
Interface Comparavel — Ordenando Objetos
Nível 2

Crie a interface Comparavel com o método int comparar(Object outro). Implemente-a na classe Produto (com nome e preco): retorne negativo se este produto for mais barato, zero se igual, positivo se mais caro. Use essa interface para encontrar o produto mais barato de um array.

                
 Solução

                public interface Comparavel {
    int comparar(Object outro);
}

public class Produto implements Comparavel {
    private String nome;
    private double preco;

    public Produto(String nome, double preco) {
        this.nome = nome; this.preco = preco;
    }

    @Override
    public int comparar(Object outro) {
        Produto p = (Produto) outro;
        return Double.compare(this.preco, p.preco);
    }

    public String toString() {
        return nome + " — R$ " + preco;
    }
}

// Encontrando o mais barato
Produto[] estoque = {
    new Produto("Teclado",  150.0),
    new Produto("Mouse",    80.0),
    new Produto("Monitor",  900.0)
};

Produto maisBarato = estoque[0];
for (Produto p : estoque) {
    if (p.comparar(maisBarato) < 0) maisBarato = p;
}
System.out.println("Mais barato: " + maisBarato);
// Mais barato: Mouse — R$ 80.0
04
Sistema de Funcionários — Herança + Interface + Polimorfismo
Nível 3

Crie a interface Tributavel com double calcularImposto(). Crie a superclasse abstrata Funcionario com nome e salarioBase e o método abstrato calcularSalario(). Implemente:

  • Horista — salário = horasTrabalhadas × valorHora
  • Assalariado — salário = salarioBase + bonus. Implementa Tributavel: imposto = 15% se salário > 3000
                
 Solução

                public interface Tributavel {
    double calcularImposto();
}

public abstract class Funcionario {
    protected String nome;
    protected double salarioBase;

    public Funcionario(String nome, double sal) {
        this.nome = nome; this.salarioBase = sal;
    }
    public abstract double calcularSalario();

    public void exibirFicha() {
        System.out.println("Funcionário: " + nome
            + " | Salário: R$ " + String.format("%.2f", calcularSalario()));
    }
}

class Horista extends Funcionario {
    private int    horas;
    private double valorHora;

    public Horista(String nome, int horas, double valorHora) {
        super(nome, 0);
        this.horas = horas; this.valorHora = valorHora;
    }
    @Override public double calcularSalario() { return horas * valorHora; }
}

class Assalariado extends Funcionario implements Tributavel {
    private double bonus;

    public Assalariado(String nome, double sal, double bonus) {
        super(nome, sal); this.bonus = bonus;
    }
    @Override public double calcularSalario() { return salarioBase + bonus; }
    @Override public double calcularImposto() {
        double sal = calcularSalario();
        return sal > 3000 ? sal * 0.15 : 0;
    }
}

// Teste
Funcionario[] equipe = {
    new Horista("Ana", 160, 25.0),
    new Assalariado("Bruno", 3500.0, 500.0)
};

for (Funcionario f : equipe) {
    f.exibirFicha();
    if (f instanceof Tributavel t) {
        System.out.println("  Imposto: R$ "
            + String.format("%.2f", t.calcularImposto()));
    }
}
              
Funcionário: Ana | Salário: R$ 4000.00
Funcionário: Bruno | Salário: R$ 4000.00
  Imposto: R$ 600.00

Resumo da Unidade

O que você aprendeu nesta unidade:

  • Herança (extends) permite que uma subclasse herde atributos e métodos da superclasse, evitando repetição. super() chama o construtor da superclasse.
  • protected torna membros visíveis para subclasses sem abrir ao mundo externo.
  • Sobrescrita (@Override) redefine um método herdado na subclasse com o mesmo nome e parâmetros — comportamento específico para cada tipo.
  • Polimorfismo permite tratar objetos de tipos diferentes de forma uniforme usando uma variável do tipo da superclasse — a JVM resolve qual método chamar em tempo de execução.
  • Classes abstratas não podem ser instanciadas diretamente e podem ter métodos abstratos (sem implementação) que as subclasses devem implementar.
  • Interfaces são contratos que definem comportamentos sem estado. Uma classe pode implementar múltiplas interfaces com implements.
  • Acoplamento dinâmico (late binding) é o mecanismo da JVM de decidir em tempo de execução qual versão de método sobrescrito invocar, com base no tipo real do objeto.
