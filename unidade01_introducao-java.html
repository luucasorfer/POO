1. O que é Java?

Java é uma das linguagens de programação mais populares e utilizadas no mundo. É uma linguagem de alto nível, orientada a objetos e portável — o que significa que o mesmo código pode ser executado em diferentes sistemas operacionais sem modificações.

Criada por James Gosling em 1991 na Sun Microsystems, Java foi projetada com o princípio "Write Once, Run Anywhere" (Escreva uma vez, execute em qualquer lugar).

Como Java funciona? A JVM

Quando você escreve e compila um programa Java, o código não é convertido diretamente para instruções da máquina. Em vez disso, ele é transformado em um formato intermediário chamado bytecode, que é executado pela Java Virtual Machine (JVM).

Você escreve
Main.java
Compilador
javac
Bytecode
Main.class
Executa
JVM

A JVM também gerencia a memória automaticamente por meio do Garbage Collector, que elimina objetos que não estão mais em uso — você não precisa se preocupar com isso no início.

Onde Java é utilizado?

  • Desenvolvimento web: Frameworks como Spring para aplicações robustas
  • Aplicativos Android: O sistema operacional móvel mais usado do mundo tem base em Java
  • Sistemas empresariais: Bancos, seguradoras e grandes corporações usam Java pela segurança e escalabilidade
  • Dispositivos embarcados: TVs inteligentes, terminais de pagamento e impressoras

3. Estrutura Básica de um Programa Java

Todo programa Java é organizado em classes e métodos. Veja o programa mais simples possível: 

          

             Main.java — Olá, Mundo!
            

              

              

              

            

          

          public class Main {

    public static void main(String[] args) {

        System.out.println("Olá, mundo!");

    }

}

Analisando cada parte:

  • public class Main — define a classe. O nome do arquivo deve ser Main.java
  • public static void main(String[] args) — ponto de entrada do programa. É sempre por aqui que a execução começa
  • System.out.println(...) — imprime uma linha de texto no console

Java é sensível a maiúsculas e minúsculas. Main é diferente de main e de MAIN. Sempre preste atenção à capitalização.

4. Tipos de Dados

Java é uma linguagem fortemente tipada: toda variável precisa ter um tipo definido antes de ser usada. O tipo informa ao Java qual tipo de dado aquela variável pode armazenar.

Tipos primitivos principais

Tipo Exemplo de valor Descrição Tamanho
int 10, -5, 0 Números inteiros 32 bits
double 10.5, 3.14 Números decimais (ponto flutuante) 64 bits
float 0.5f, 1.0f Decimal com menos precisão — requer sufixo f 32 bits
char 'A', 'z' Um único caractere — usa aspas simples 16 bits
boolean true / false Verdadeiro ou falso 1 bit
String "Olá", "Java" Texto (não é primitivo, é uma classe) — usa aspas duplas variável

String começa com letra maiúscula porque é uma classe, não um tipo primitivo. Mesmo assim, é o tipo mais usado para texto em Java.

5. Variáveis

Variáveis são espaços na memória usados para armazenar dados. Em Java, toda variável precisa ter um tipo e um nome. 

          

             Declarando e inicializando variáveis
            

              

              

              

            

          

          int     idade   = 20;          // número inteiro
double  altura  = 1.75;        // número decimal
float   peso    = 70.5f;       // decimal (sufixo f obrigatório)
char    letra   = 'A';          // um caractere (aspas simples)
boolean ativo   = true;         // verdadeiro ou falso
String  nome    = "Lucas";      // texto (aspas duplas)

Regras para nomear variáveis

  • Devem ter um tipo definido
  • O nome não pode começar com número — mas pode conter números
  • Sem espaços — use camelCase para nomes compostos
  • Java é sensível a maiúsculas: idade ≠ Idade
  • Não use palavras reservadas como class, int, void
          

             Exemplos corretos e incorretos
            

              

              

              

            

          

          // ✅ Corretos
int idade = 18;
int idadeDoAluno = 20;
double precoFinal = 99.90;

// ❌ Incorretos
// int 2idade = 18;       — não pode começar com número
// int idade aluno = 18; — não pode ter espaço
// int class = 18;       — palavra reservada

6. Entrada e Saída de Dados

Saída de dados — System.out

Para exibir informações no console, usamos System.out. Há duas variações:

          

             Saída no console
            

              

              

              

            

          

          // println — imprime e quebra a linha
System.out.println("Olá, mundo!");
System.out.println("Segunda linha");

// print — imprime sem quebrar a linha
System.out.print("A");
System.out.print("B");   // imprime: AB

// Exibindo variáveis
int idade = 20;
System.out.println("Idade: " + idade);  // Idade: 20

Entrada de dados — Scanner

Para ler dados digitados pelo usuário no console, usamos a classe Scanner. Ela precisa ser importada antes de ser usada: 

          

             Lendo dados com Scanner
            

              

              

              

            

          

          import java.util.Scanner;  // importar no topo do arquivo

public class Main {
    public static void main(String[] args) {

        Scanner sc = new Scanner(System.in);

        System.out.print("Digite seu nome: ");
        String nome = sc.nextLine();

        System.out.print("Digite sua idade: ");
        int idade = sc.nextInt();

        System.out.println("Olá, " + nome + "! Você tem " + idade + " anos.");

        sc.close();  // boa prática: fechar o Scanner ao fim
    }
}
MétodoLêExemplo
nextLine()Uma linha de texto completasc.nextLine()
nextInt()Um número inteirosc.nextInt()
nextDouble()Um número decimalsc.nextDouble()
next()Uma palavra (sem espaço)sc.next()

Operadores são símbolos que realizam operações sobre valores e variáveis.

Aritméticos
+Adição
-Subtração
*Multiplicação
/Divisão
%Resto (módulo)
Relacionais
==Igual a
!=Diferente de
>Maior que
<Menor que
>=Maior ou igual
<=Menor ou igual
Lógicos
&&E (AND)
||OU (OR)
!NÃO (NOT)
          

             Operadores em uso
            

              

              

              

            

          

          int a = 10, b = 3;

// Aritméticos
System.out.println(a + b);   // 13
System.out.println(a / b);   // 3  (divisão inteira)
System.out.println(a % b);   // 1  (resto de 10/3)

// Relacionais — retornam true ou false
System.out.println(a > b);   // true
System.out.println(a == b);  // false

// Lógicos — combinam condições
boolean maior = a > 5 && b > 1;  // true (ambas verdadeiras)
boolean algum = a > 20 || b > 1; // true (pelo menos uma)

8. Estruturas Condicionais

Estruturas condicionais permitem que o programa tome decisões com base em condições. O fluxo de execução muda dependendo se a condição é verdadeira ou falsa.

if / else if / else

          

             if / else if / else
            

              

              

              

            

          

          int idade = 16;

if (idade >= 18) {
    System.out.println("Maior de idade");
} else if (idade >= 16) {
    System.out.println("Quase lá — 16 ou 17 anos");
} else {
    System.out.println("Menor de idade");
}

// Saída: Quase lá — 16 ou 17 anos

switch

O switch é ideal quando você precisa comparar uma variável com múltiplos valores fixos: 

          

             switch / case
            

              

              

              

            

          

          int dia = 3;

switch (dia) {
    case 1:
        System.out.println("Segunda-feira");
        break;
    case 2:
        System.out.println("Terça-feira");
        break;
    case 3:
        System.out.println("Quarta-feira");
        break;
    default:
        System.out.println("Outro dia");
}

// Saída: Quarta-feira

Não esqueça o break em cada case! Sem ele, a execução continua para o próximo case mesmo que a condição já tenha sido satisfeita.

9. Estruturas de Repetição

Estruturas de repetição (ou laços) permitem executar um bloco de código várias vezes com base em uma condição.

for — quantidade definida de repetições

Use for quando você sabe quantas vezes o bloco deve repetir:

          

             Laço for
            

              

              

              

            

          

          //  início  condição  incremento
for (int i = 0; i < 5; i++) {
    System.out.println("Iteração: " + i);
}

// Saída:
// Iteração: 0
// Iteração: 1
// Iteração: 2
// Iteração: 3
// Iteração: 4

while — repete enquanto a condição for verdadeira

Use while quando não sabe quantas vezes o laço vai repetir — depende de uma condição:

          

             Laço while
            

              

              

              

            

          

          int i = 0;

while (i < 5) {
    System.out.println("Valor: " + i);
    i++;  // incrementa i — sem isso, loop infinito!
}

do-while — executa pelo menos uma vez

O do-while garante que o bloco seja executado ao menos uma vez, mesmo que a condição seja falsa desde o início:

          

             Laço do-while
            

              

              

              

            

          

          int i = 0;

do {
    System.out.println("Executado: " + i);
    i++;
} while (i < 3);  // verifica a condição ao final

Resumo: qual laço usar?

  • for — quando sabe a quantidade de repetições
  • while — quando repete até uma condição ser falsa
  • do-while — quando precisa executar pelo menos uma vez

10. Convenções de Nomenclatura

Convenções de nomenclatura são recomendações para nomear variáveis, classes e métodos de forma padronizada. Seguir essas convenções torna o código mais legível e profissional.

Os quatro padrões mais comuns são:

camelCase
idadeDoAluno
valorFinal
Primeira palavra minúscula, demais com inicial maiúscula.
PascalCase
IdadeDoAluno
ContaBancaria
Todas as palavras com inicial maiúscula.
SCREAMING_SNAKE
VALOR_MAXIMO
PI
Tudo maiúsculo com underline. Usado em constantes.
snake_case
idade_do_aluno
valor_final
Tudo minúsculo com underline. Não é comum em Java.

Convenções específicas do Java

Java define convenções claras para cada tipo de identificador:

O que é Padrão Exemplo
Variáveis camelCase idadeDoAluno, precoFinal
Métodos camelCase calcularMedia(), getNome()
Classes PascalCase ContaBancaria, Pessoa
Constantes SCREAMING_SNAKE VALOR_MAXIMO, PI
Pacotes minúsculas br.com.escola, util 
          

             Convenções aplicadas na prática
            

              

              

              

            

          

          // Classe — PascalCase
public class ContaBancaria {

    // Constante — SCREAMING_SNAKE_CASE
    static final double TAXA_JUROS = 0.05;

    // Atributo — camelCase
    private double saldoAtual;

    // Método — camelCase
    public void calcularRendimento() {
        saldoAtual += saldoAtual * TAXA_JUROS;
    }

}

Seguir as convenções não é obrigatório para o programa funcionar, mas é essencial para trabalhar em equipe e manter o código organizado e legível.

Resumo da Unidade

O que você aprendeu nesta unidade:

  • Java é uma linguagem portável — o bytecode é executado pela JVM em qualquer sistema operacional.
  • Para programar em Java precisamos do JDK (compilador) e de uma IDE como o Eclipse.
  • Todo programa Java começa pela classe e pelo método public static void main(String[] args).
  • Java é fortemente tipado — toda variável precisa de um tipo (int, double, String…).
  • System.out.println() exibe dados no console; Scanner lê dados do usuário.
  • Operadores: aritméticos (+, -, *, /), relacionais (==, !=, >, <) e lógicos (&&, ||, !).
  • if/else e switch controlam o fluxo condicional do programa.
  • for, while e do-while repetem blocos de código.
  • Convenções Java: camelCase para variáveis e métodos, PascalCase para classes, SCREAMING_SNAKE para constantes.
