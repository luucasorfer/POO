1. Swing vs JavaFX — Qual usar?

Java oferece duas bibliotecas principais para construção de interfaces gráficas. Nesta unidade usaremos Swing, que já vem incluído no JDK sem nenhuma configuração extra.

☕ Swing
  • Incluso no JDK — zero dependências extras
  • Maduro e estável — usado há décadas em produção
  • Visual nativo do sistema operacional
  • Aprende mais rápido — ideal para início
  • Estilo visual mais simples e datado
🎨 JavaFX
  • Visual moderno com CSS e animações
  • FXML — separa interface de lógica
  • Suporte a multimídia — vídeo, áudio, web
  • Futuro do Java GUI
  • Requer dependência separada desde Java 11

Nesta unidade usamos Swing. Os conceitos fundamentais — componentes, eventos, layout — são os mesmos em JavaFX. Aprender Swing bem facilita muito a transição.

2. Criando a Primeira Janela

No Swing, a janela principal é um JFrame. Ele é o contêiner raiz da aplicação — todos os outros componentes são adicionados dentro dele.

Minha Primeira Janela
janela vazia — 400 × 300 px 
          
 PrimeiraJanela.java — JFrame básico

          import javax.swing.JFrame;
import javax.swing.SwingUtilities;

public class PrimeiraJanela {

    public static void main(String[] args) {

        // Boa prática: criar a GUI na Event Dispatch Thread
        SwingUtilities.invokeLater(() -> {

            JFrame janela = new JFrame("Minha Primeira Janela");

            janela.setSize(400, 300);              // largura x altura
            janela.setLocationRelativeTo(null);    // centraliza na tela
            janela.setDefaultCloseOperation(
                JFrame.EXIT_ON_CLOSE);              // fecha ao clicar no X
            janela.setVisible(true);              // exibe a janela

        });
    }
}

SwingUtilities.invokeLater() — o Swing não é thread-safe. Toda criação e modificação de componentes deve ocorrer na Event Dispatch Thread (EDT). Usar invokeLater garante isso. Sempre faça isso em aplicações reais.

Configurações essenciais do JFrame

Método O que faz
setTitle("Texto") Define o título da barra superior
setSize(w, h) Define largura e altura em pixels
setResizable(false) Impede o redimensionamento pelo usuário
setLocationRelativeTo(null) Centraliza a janela na tela
setDefaultCloseOperation(...) Define o que acontece ao fechar: EXIT_ON_CLOSE, DISPOSE_ON_CLOSE, DO_NOTHING_ON_CLOSE
pack() Ajusta o tamanho ao conteúdo automaticamente
setVisible(true) Torna a janela visível — deve ser o último método chamado

3. Componentes Básicos

Os componentes são os elementos visuais que compõem a interface: botões, campos de texto, rótulos, listas etc. No Swing todos começam com a letra J.

JLabel
Rótulo
Exibe texto ou imagem estático — não editável pelo usuário.
JTextField
Campo de texto
Entrada de texto em uma única linha.
JPasswordField
Campo de senha
Campo de texto que oculta os caracteres digitados.
JTextArea
Área de texto
Campo de texto com múltiplas linhas.
JButton
Botão
Clicável, dispara uma ação quando pressionado.
JCheckBox
Caixa de seleção
Opção que pode ser marcada ou desmarcada.
JRadioButton
Botão de rádio
Seleção exclusiva dentro de um grupo.
JComboBox
Lista suspensa
Menu dropdown com opções predefinidas.
JPanel
Painel
Contêiner genérico para agrupar e organizar outros componentes.

Criando e adicionando componentes 

          
 Adicionando componentes ao JFrame

          import javax.swing.*;
import java.awt.*;

public class JanelaComComponentes {

    public static void main(String[] args) {
        SwingUtilities.invokeLater(() -> {

            JFrame     janela = new JFrame("Exemplo");
            JLabel     label  = new JLabel("Nome:");
            JTextField campo  = new JTextField(20);  // 20 colunas
            JButton    botao  = new JButton("Enviar");

            // Painel com FlowLayout (componentes em linha)
            JPanel painel = new JPanel();
            painel.add(label);
            painel.add(campo);
            painel.add(botao);

            janela.add(painel);
            janela.pack();
            janela.setLocationRelativeTo(null);
            janela.setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
            janela.setVisible(true);

        });
    }
}

4. Layout Managers

Os Layout Managers controlam automaticamente como os componentes são posicionados e redimensionados dentro de um contêiner. Em vez de definir coordenadas fixas, o layout adapta a interface a diferentes resoluções.

FlowLayout
Fluxo horizontal
Componentes em linha, quebra quando necessário. Padrão do JPanel.
BorderLayout
Cinco regiões
Divide em NORTH, SOUTH, EAST, WEST, CENTER. Padrão do JFrame.
GridLayout
Grade regular
Divide em N linhas × M colunas de tamanho igual.
BoxLayout
Coluna ou linha
Empilha componentes vertical ou horizontalmente com controle de alinhamento.
GridBagLayout
Grade flexível
O mais poderoso — permite células de tamanhos diferentes e spanning.
null Layout
Posição absoluta
Posicionamento manual com setBounds(). Evite — quebra com DPI diferente. 
          
 BorderLayout — o mais usado no JFrame

          JFrame f = new JFrame("BorderLayout");
f.setLayout(new BorderLayout());  // já é o padrão do JFrame

f.add(new JButton("Norte"),  BorderLayout.NORTH);
f.add(new JButton("Sul"),    BorderLayout.SOUTH);
f.add(new JButton("Leste"),   BorderLayout.EAST);
f.add(new JButton("Oeste"),   BorderLayout.WEST);
f.add(new JTextArea(),        BorderLayout.CENTER);
        
          
 GridLayout — formulário em grade 4×2

          JPanel form = new JPanel(new GridLayout(4, 2, 8, 6));
//                                           linhas  colunas  hgap  vgap

form.add(new JLabel("Nome:"));    form.add(new JTextField());
form.add(new JLabel("Idade:"));   form.add(new JTextField());
form.add(new JLabel("Email:"));   form.add(new JTextField());
form.add(new JLabel());          form.add(new JButton("Cadastrar"));

5. Eventos e Listeners

Eventos são ações do usuário: clicar em um botão, digitar texto, mover o mouse, fechar a janela. Para responder a esses eventos, usamos Listeners — objetos que "escutam" e executam código quando o evento ocorre.

Modelo de eventos do Swing:

  • Fonte do evento — o componente que dispara (ex: JButton)
  • Listener — quem escuta e reage ao evento
  • Registro — conectar o listener à fonte com addActionListener(...)

ActionListener — o evento mais usado

O ActionListener captura cliques em botões, itens de menu e tecla Enter em campos de texto: 

          
 ActionListener com Lambda (Java 8+)

          JButton btn = new JButton("Clique aqui");

// Registrando o listener com lambda — forma moderna e concisa
btn.addActionListener(e -> {
    System.out.println("Botão clicado!");
});

// Forma equivalente sem lambda (antes do Java 8):
btn.addActionListener(new ActionListener() {
    @Override
    public void actionPerformed(ActionEvent e) {
        System.out.println("Botão clicado!");
    }
});

Principais tipos de evento

Listener Quando usar Método principal
ActionListener Clique em botão, Enter em campo, item de menu actionPerformed()
MouseListener Clique, pressionamento, soltura, entrada/saída do mouse mouseClicked()
KeyListener Tecla pressionada, solta ou digitada keyPressed()
FocusListener Componente ganha ou perde o foco focusGained()
WindowListener Janela abrindo, fechando, minimizando windowClosing()
DocumentListener Texto de um campo muda em tempo real insertUpdate()

Lendo valores dos componentes 

          
 Lendo e escrevendo valores em componentes

          JTextField  txtNome  = new JTextField(20);
JLabel      lblSaida = new JLabel("Resultado aparece aqui");
JButton     btn      = new JButton("Confirmar");

btn.addActionListener(e -> {

    // lendo o texto do campo
    String nome = txtNome.getText().trim();

    // escrevendo no label
    lblSaida.setText("Olá, " + nome + "!");

    // limpando o campo após a leitura
    txtNome.setText("");
    txtNome.requestFocus();  // devolve o foco ao campo

});

6. Tratamento de Exceções na Interface Gráfica

Em aplicações gráficas o tratamento de exceções é ainda mais importante: uma exceção não tratada dentro de um listener não fecha o programa, mas pode deixar a interface num estado inconsistente. O correto é capturar o erro e exibir uma mensagem amigável ao usuário.

JOptionPane — diálogos prontos

O JOptionPane é a forma mais rápida de exibir mensagens, erros e pedir confirmações em janelas gráficas: 

          
 JOptionPane — tipos de diálogo

          // ── Mensagem de informação ──────────────────────────
JOptionPane.showMessageDialog(
    janela,
    "Operação realizada com sucesso!",
    "Sucesso",
    JOptionPane.INFORMATION_MESSAGE
);

// ── Mensagem de erro ────────────────────────────────
JOptionPane.showMessageDialog(
    janela,
    "Valor inválido. Digite apenas números.",
    "Erro de entrada",
    JOptionPane.ERROR_MESSAGE
);

// ── Confirmação ─────────────────────────────────────
int resp = JOptionPane.showConfirmDialog(
    janela,
    "Tem certeza que deseja sair?",
    "Confirmar",
    JOptionPane.YES_NO_OPTION
);
if (resp == JOptionPane.YES_OPTION) {
    System.exit(0);
}

try/catch dentro de listeners

Toda operação que pode falhar dentro de um listener deve ser protegida com try/catch. O erro deve ser comunicado ao usuário via JOptionPane, nunca silenciado: 

          
 Validação com try/catch no listener

          JTextField txtIdade  = new JTextField(10);
JLabel     lblResult = new JLabel(" ");
JButton    btnCalc   = new JButton("Verificar");

btnCalc.addActionListener(e -> {
    try {
        String texto = txtIdade.getText().trim();
        int idade  = Integer.parseInt(texto);  // pode lançar NumberFormatException

        if (idade < 0 || idade > 150) {
            throw new IllegalArgumentException("Idade fora do intervalo válido.");
        }

        String msg = idade >= 18
            ? "✅ Maior de idade"
            : "⚠️ Menor de idade";
        lblResult.setText(msg);

    } catch (NumberFormatException ex) {
        JOptionPane.showMessageDialog(
            null,
            "Digite um número inteiro válido.",
            "Entrada inválida",
            JOptionPane.WARNING_MESSAGE
        );
        txtIdade.setText("");
        txtIdade.requestFocus();

    } catch (IllegalArgumentException ex) {
        JOptionPane.showMessageDialog(
            null,
            ex.getMessage(),
            "Valor inválido",
            JOptionPane.ERROR_MESSAGE
        );
    }
});

7. Padrão: Classe que Estende JFrame

À medida que a interface cresce, organizar tudo no main fica inviável. A prática padrão é criar uma classe própria que estende JFrame, centralizando a criação dos componentes e os listeners num único lugar. 

          
 Calculadora.java — classe que estende JFrame

          import javax.swing.*;
import java.awt.*;

public class Calculadora extends JFrame {

    // ── Atributos — componentes da tela ─────────────────
    private JTextField txtA;
    private JTextField txtB;
    private JLabel     lblResultado;
    private JButton    btnSomar;
    private JButton    btnDividir;

    // ── Construtor — monta a interface ──────────────────
    public Calculadora() {
        setTitle("Calculadora");
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);

        inicializarComponentes();
        registrarEventos();

        pack();
        setLocationRelativeTo(null);
        setVisible(true);
    }

    // ── Criação dos componentes ─────────────────────────
    private void inicializarComponentes() {
        txtA        = new JTextField(10);
        txtB        = new JTextField(10);
        btnSomar   = new JButton("Somar");
        btnDividir = new JButton("Dividir");
        lblResultado = new JLabel("Resultado: ---");
        lblResultado.setFont(new Font("Monospaced", Font.BOLD, 14));

        JPanel pEntrada = new JPanel(new GridLayout(2, 2, 8, 6));
        pEntrada.add(new JLabel("Valor A:")); pEntrada.add(txtA);
        pEntrada.add(new JLabel("Valor B:")); pEntrada.add(txtB);

        JPanel pBotoes = new JPanel(new FlowLayout());
        pBotoes.add(btnSomar);
        pBotoes.add(btnDividir);

        JPanel pPrincipal = new JPanel(new BorderLayout(10, 10));
        pPrincipal.setBorder(
            BorderFactory.createEmptyBorder(12, 12, 12, 12));
        pPrincipal.add(pEntrada,     BorderLayout.NORTH);
        pPrincipal.add(pBotoes,      BorderLayout.CENTER);
        pPrincipal.add(lblResultado, BorderLayout.SOUTH);

        add(pPrincipal);
    }

    // ── Registro dos listeners ──────────────────────────
    private void registrarEventos() {

        btnSomar.addActionListener(e -> {
            try {
                double a = Double.parseDouble(txtA.getText());
                double b = Double.parseDouble(txtB.getText());
                lblResultado.setText("Resultado: " + (a + b));
            } catch (NumberFormatException ex) {
                mostrarErro("Digite números válidos nos dois campos.");
            }
        });

        btnDividir.addActionListener(e -> {
            try {
                double a = Double.parseDouble(txtA.getText());
                double b = Double.parseDouble(txtB.getText());
                if (b == 0)
                    throw new ArithmeticException("Divisor não pode ser zero.");
                lblResultado.setText("Resultado: " + (a / b));
            } catch (NumberFormatException ex) {
                mostrarErro("Digite números válidos nos dois campos.");
            } catch (ArithmeticException ex) {
                mostrarErro(ex.getMessage());
            }
        });
    }

    // ── Método auxiliar para erros ──────────────────────
    private void mostrarErro(String msg) {
        JOptionPane.showMessageDialog(
            this, msg, "Erro", JOptionPane.ERROR_MESSAGE);
    }

    // ── Ponto de entrada ────────────────────────────────
    public static void main(String[] args) {
        SwingUtilities.invokeLater(Calculadora::new);
    }

}

8. Aplicação Completa — Cadastro de Alunos

Vamos construir uma aplicação completa que integra todos os conceitos da unidade: componentes, layout, eventos, validação com try/catch e exibição de resultados. O sistema permite cadastrar alunos e exibir a lista num JTextArea.

Cadastro de Alunos
Nome:
Ana Lima
Nota (0-10):
8.5
Cadastrar
Limpar Lista
[ ALUNOS CADASTRADOS ] 1. Ana Lima — Nota: 8.5 | ✅ Aprovado 2. Pedro Costa — Nota: 4.2 | ❌ Reprovado 
          
 CadastroAlunos.java — aplicação completa

          import javax.swing.*;
import java.awt.*;
import java.util.ArrayList;

public class CadastroAlunos extends JFrame {

    // ── Componentes ─────────────────────────────────────
    private JTextField txtNome;
    private JTextField txtNota;
    private JButton    btnCadastrar;
    private JButton    btnLimpar;
    private JTextArea  areaLista;

    // ── Dados ────────────────────────────────────────────
    private ArrayList<String> alunos = new ArrayList<>();

    // ── Construtor ───────────────────────────────────────
    public CadastroAlunos() {
        setTitle("Cadastro de Alunos");
        setDefaultCloseOperation(JFrame.EXIT_ON_CLOSE);
        inicializar();
        registrar();
        pack();
        setLocationRelativeTo(null);
        setVisible(true);
    }

    // ── Montagem da interface ────────────────────────────
    private void inicializar() {
        txtNome      = new JTextField(18);
        txtNota      = new JTextField(6);
        btnCadastrar = new JButton("Cadastrar");
        btnLimpar    = new JButton("Limpar Lista");
        areaLista    = new JTextArea(10, 32);
        areaLista.setEditable(false);
        areaLista.setFont(new Font("Monospaced", Font.PLAIN, 12));
        areaLista.setText("[ ALUNOS CADASTRADOS ]\n");

        // Formulário de entrada
        JPanel pForm = new JPanel(new GridLayout(2, 2, 8, 6));
        pForm.add(new JLabel("Nome:"));         pForm.add(txtNome);
        pForm.add(new JLabel("Nota (0-10):"));  pForm.add(txtNota);

        // Botões
        JPanel pBotoes = new JPanel(new FlowLayout(FlowLayout.RIGHT));
        pBotoes.add(btnLimpar);
        pBotoes.add(btnCadastrar);

        // Layout principal
        JPanel pPrincipal = new JPanel(new BorderLayout(10, 10));
        pPrincipal.setBorder(
            BorderFactory.createEmptyBorder(12, 12, 12, 12));
        pPrincipal.add(pForm,    BorderLayout.NORTH);
        pPrincipal.add(pBotoes,  BorderLayout.CENTER);
        pPrincipal.add(new JScrollPane(areaLista),
                         BorderLayout.SOUTH);
        add(pPrincipal);
    }

    // ── Eventos ──────────────────────────────────────────
    private void registrar() {

        btnCadastrar.addActionListener(e -> {
            try {
                String nome = txtNome.getText().trim();
                if (nome.isEmpty())
                    throw new IllegalArgumentException(
                        "O campo Nome não pode estar vazio.");

                double nota = Double.parseDouble(txtNota.getText());
                if (nota < 0 || nota > 10)
                    throw new IllegalArgumentException(
                        "A nota deve estar entre 0 e 10.");

                String situacao = nota >= 6.0
                    ? "✅ Aprovado" : "❌ Reprovado";

                alunos.add(nome + " — Nota: " + nota
                    + " | " + situacao);
                atualizarLista();

                txtNome.setText("");
                txtNota.setText("");
                txtNome.requestFocus();

            } catch (NumberFormatException ex) {
                mostrarErro("Nota inválida. Use ponto ou vírgula: ex 7.5");
            } catch (IllegalArgumentException ex) {
                mostrarErro(ex.getMessage());
            }
        });

        btnLimpar.addActionListener(e -> {
            int resp = JOptionPane.showConfirmDialog(
                this, "Limpar toda a lista?",
                "Confirmar", JOptionPane.YES_NO_OPTION);
            if (resp == JOptionPane.YES_OPTION) {
                alunos.clear();
                atualizarLista();
            }
        });

        // Enter no campo de nota aciona o cadastro
        txtNota.addActionListener(
            btnCadastrar.getActionListeners()[0]);
    }

    // ── Helpers ───────────────────────────────────────────
    private void atualizarLista() {
        StringBuilder sb = new StringBuilder("[ ALUNOS CADASTRADOS ]\n");
        for (int i = 0; i < alunos.size(); i++)
            sb.append((i+1) + ". " + alunos.get(i) + "\n");
        areaLista.setText(sb.toString());
    }

    private void mostrarErro(String msg) {
        JOptionPane.showMessageDialog(
            this, msg, "Erro", JOptionPane.ERROR_MESSAGE);
    }

    public static void main(String[] args) {
        SwingUtilities.invokeLater(CadastroAlunos::new);
    }

}

9. Boas Práticas em Swing

Sempre use SwingUtilities.invokeLater() Toda criação e atualização de componentes Swing deve ocorrer na Event Dispatch Thread. Ignorar isso causa comportamentos imprevisíveis e bugs difíceis de reproduzir.
Separe a construção da interface da lógica Use métodos como inicializarComponentes() e registrarEventos() para manter o construtor limpo e o código organizado.
Nunca deixe exceções silenciosas em listeners Um catch vazio ou que só imprime no console deixa o usuário sem entender o que aconteceu. Use sempre JOptionPane para comunicar erros.
Prefira pack() a setSize() fixo pack() calcula o tamanho ideal para o conteúdo, adaptando-se a fontes e resoluções diferentes. setSize() com valores fixos quebra em telas com DPI alto.
Envolva JTextArea em JScrollPane Se o texto ultrapassar o espaço visível, o usuário não consegue rolar sem JScrollPane. Use sempre new JScrollPane(areaTexto) ao adicionar ao contêiner.
Use BorderFactory para margens internas BorderFactory.createEmptyBorder(top, left, bottom, right) adiciona espaçamento interno aos painéis, deixando a interface mais arejada e profissional.

Resumo da Unidade

O que você aprendeu nesta unidade:

  • Swing é a biblioteca de GUI padrão do Java — já inclusa no JDK, sem dependências extras.
  • JFrame é a janela principal. Configure com setTitle, setDefaultCloseOperation, pack() e setVisible(true).
  • Sempre crie a GUI dentro de SwingUtilities.invokeLater() para garantir execução na Event Dispatch Thread.
  • Componentes como JLabel, JTextField, JButton, JTextArea e JPanel são a base de toda interface Swing.
  • Layout Managers (FlowLayout, BorderLayout, GridLayout) posicionam os componentes de forma adaptável.
  • Listeners como ActionListener respondem a eventos do usuário. Use lambdas para simplificar.
  • Todo código que pode lançar exceção dentro de um listener deve estar em try/catch, com erros comunicados via JOptionPane.
  • Organizar a janela em uma classe que estende JFrame com métodos separados para criação e eventos torna o código limpo e manutenível.
