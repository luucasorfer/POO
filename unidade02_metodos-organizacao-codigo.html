1. O que é um Método?

Um método é um bloco de código com um nome, que pode ser chamado quantas vezes quisermos para executar uma tarefa específica. Em vez de repetir as mesmas linhas de código, você escreve uma vez e reutiliza sempre que precisar.

🔁

Reutilização

Escreva uma vez, chame quantas vezes precisar.

📦

Organização

Divide o programa em partes menores e mais legíveis.

🔧

Manutenção

Corrigir um método corrige todos os pontos que o chamam.

🎯

Abstração

Você usa o método sem precisar saber como ele funciona por dentro.

Analogia: um método é como uma receita de culinária. Você não repete os passos toda vez que faz bolo — você segue a receita. O método é a receita; chamá-lo é seguir os passos.

Anatomia de um método

Todo método em Java segue a mesma estrutura:

public static  int  somar ( int  a ,  int  b ) {
return a + b;
}
Modificadores
Tipo de retorno
Nome do método
Parâmetros

Seu primeiro método

O método mais simples não recebe parâmetros e não retorna nada. Ele apenas executa uma ação. O tipo de retorno nesses casos é void — que significa "sem retorno". 

          

             Método void — sem parâmetros e sem retorno
            

              

              

              

            

          

          public class Main {

    // Definição do método
    static void exibirBoasVindas() {
        System.out.println("===========================");
        System.out.println("  Bem-vindo ao sistema!  ");
        System.out.println("===========================");
    }

    public static void main(String[] args) {
        exibirBoasVindas();  // chamando o método
        exibirBoasVindas();  // pode chamar quantas vezes quiser
    }

}

Saída (aparece duas vezes):

===========================
  Bem-vindo ao sistema!  
===========================

2. Parâmetros

Parâmetros são as informações que você passa para um método no momento de chamá-lo. Eles deixam o método mais flexível, pois o mesmo código pode trabalhar com dados diferentes a cada chamada.

Parâmetro vs Argumento

  • Parâmetro — a variável declarada na definição do método: int a
  • Argumento — o valor real passado na chamada: somar(3, 5)

Na prática, os dois termos são usados como sinônimos.

Método com um parâmetro

          

             Um parâmetro
            

              

              

              

            

          

          static void saudar(String nome) {
    System.out.println("Olá, " + nome + "!");
}

// Chamadas com argumentos diferentes
saudar("Lucas");   // Olá, Lucas!
saudar("Maria");   // Olá, Maria!
saudar("Carlos");  // Olá, Carlos!

Múltiplos parâmetros

Um método pode receber quantos parâmetros forem necessários — separados por vírgula, cada um com seu tipo:

          

             Múltiplos parâmetros
            

              

              

              

            

          

          static void apresentar(String nome, int idade, String curso) {
    System.out.println("Nome: "   + nome);
    System.out.println("Idade: "  + idade);
    System.out.println("Curso: "  + curso);
}

// A ordem dos argumentos deve seguir a dos parâmetros
apresentar("Lucas", 20, "Ciência da Computação");

Os argumentos devem ser passados na mesma ordem e com os mesmos tipos dos parâmetros declarados. apresentar(20, "Lucas", ...) causaria um erro de compilação.

3. Retorno de Valores

Um método pode devolver um resultado para quem o chamou. Para isso, declaramos o tipo de retorno no lugar de void e usamos a palavra-chave return.

void — não retorna
              

                
                

                  

                  

                  

                

              

              static void mostrar(int n) {
    System.out.println(n);
    // não há return
}
int — retorna valor
              

                
                

                  

                  

                  

                

              

              static int dobro(int n) {
    return n * 2;
    // return obrigatório
}

Usando o valor retornado

O valor retornado pode ser guardado em uma variável, usado em uma expressão ou exibido diretamente:

          

             Capturando e usando o retorno
            

              

              

              

            

          

          static double calcularMedia(double n1, double n2, double n3) {
    return (n1 + n2 + n3) / 3.0;
}

public static void main(String[] args) {

    // 1. Guardando em variável
    double media = calcularMedia(7.0, 8.5, 9.0);
    System.out.println("Média: " + media);      // Média: 8.166...

    // 2. Usando diretamente no println
    System.out.println(calcularMedia(5.0, 6.0, 7.0));  // 6.0

    // 3. Usando em uma expressão
    if (calcularMedia(7.0, 8.0, 9.0) >= 7.0) {
        System.out.println("Aprovado!");
    }

}

Retorno boolean — verificações

Métodos que retornam boolean são muito úteis para verificações. Por convenção, seus nomes começam com is ou tem:

          

             Método que retorna boolean
            

              

              

              

            

          

          static boolean isMaiorDeIdade(int idade) {
    return idade >= 18;
}

static boolean isAprovado(double media) {
    return media >= 6.0;
}

// Uso direto no if
if (isMaiorDeIdade(20)) {
    System.out.println("Acesso permitido");
}

4. Sobrecarga de Métodos

Sobrecarga (ou overloading) é quando criamos vários métodos com o mesmo nome, mas com parâmetros diferentes. O Java identifica qual versão chamar com base nos argumentos fornecidos.

Para sobrecarregar um método, você deve mudar pelo menos um dos seguintes:

  • A quantidade de parâmetros
  • O tipo de um ou mais parâmetros
  • A ordem dos tipos dos parâmetros

O tipo de retorno não conta para diferenciar sobrecargas.

Exemplo — método somar sobrecarregado

          

             Sobrecarga do método somar
            

              

              

              

            

          

          // Versão 1: dois inteiros
static int somar(int a, int b) {
    return a + b;
}

// Versão 2: três inteiros
static int somar(int a, int b, int c) {
    return a + b + c;
}

// Versão 3: dois doubles
static double somar(double a, double b) {
    return a + b;
}

public static void main(String[] args) {
    System.out.println(somar(2, 3));          // chama versão 1 → 5
    System.out.println(somar(2, 3, 4));       // chama versão 2 → 9
    System.out.println(somar(1.5, 2.5));      // chama versão 3 → 4.0
}
Chamada Versão acionada Resultado
somar(2, 3) somar(int, int) 5
somar(2, 3, 4) somar(int, int, int) 9
somar(1.5, 2.5) somar(double, double) 4.0

Você já usa sobrecarga sem perceber! O próprio System.out.println() é sobrecarregado — aceita int, double, String, boolean... tudo com o mesmo nome.

5. Escopo de Variáveis

O escopo de uma variável define onde ela existe e pode ser usada dentro do programa. Uma variável só existe dentro do bloco { } em que foi declarada.

Tipos de escopo

Diagrama de escopo — o que existe onde
Classe
static int contador = 0
— visível em todos os métodos da classe
Método main
int total = 0
— existe só dentro do main
Bloco for
int i = 0
— existe só dentro do for

Variável local

Declarada dentro de um método — existe apenas enquanto o método está sendo executado:

          

             Escopo local — erro clássico
            

              

              

              

            

          

          static void calcular() {
    int resultado = 10 * 2;
    System.out.println(resultado);  // ✅ funciona — mesmo escopo
}

public static void main(String[] args) {
    calcular();
    // System.out.println(resultado);  ❌ ERRO — resultado não existe aqui!
}

Escopo de bloco

Variáveis declaradas dentro de if, for, while existem apenas dentro desses blocos:

          

             Escopo de bloco
            

              

              

              

            

          

          for (int i = 0; i < 5; i++) {
    int dobro = i * 2;     // dobro existe só dentro do for
    System.out.println(dobro);
}

// System.out.println(i);     ❌ ERRO — i só existe dentro do for
// System.out.println(dobro); ❌ ERRO — dobro só existe dentro do for

Variável de classe (static)

Declarada fora dos métodos, com static — acessível por todos os métodos da classe:

          

             Variável de classe — compartilhada
            

              

              

              

            

          

          public class Contador {

    static int total = 0;  // variável de classe

    static void incrementar() {
        total++;              // acessa a variável de classe
    }

    public static void main(String[] args) {
        incrementar();
        incrementar();
        incrementar();
        System.out.println(total);  // 3
    }

}

6. Boas Práticas de Organização

Um método, uma responsabilidade Cada método deve fazer apenas uma coisa. Se ele está fazendo muitas coisas, divida em métodos menores.
Nomes descritivos — camelCase Prefira calcularMediaFinal() a calc(). O nome deve explicar o que o método faz.
Mantenha os métodos curtos Um método muito longo é difícil de entender e testar. Se passar de 20 linhas, considere dividir.
Evite repetição de código (DRY) Don't Repeat Yourself — se você copia e cola código, isso provavelmente deveria ser um método.
Limite o número de parâmetros Mais de 3 ou 4 parâmetros dificulta o uso. Considere agrupar dados relacionados em um objeto.
Declare variáveis próximo ao uso Não declare todas as variáveis no topo do método — declare cada uma perto de onde ela será usada pela primeira vez.

7. Exercícios Práticos

Os exercícios abaixo são progressivos — cada um introduz um conceito novo que depende do anterior. Tente resolver antes de ver a solução.

01
Saudação Personalizada
Nível 1

Crie um método saudar que receba um nome e uma hora do dia (inteiro de 0 a 23) e exiba uma saudação adequada: "Bom dia", "Boa tarde" ou "Boa noite".

  • 0–11 → Bom dia
  • 12–17 → Boa tarde
  • 18–23 → Boa noite

Exemplo de chamada: saudar("Lucas", 9) → "Bom dia, Lucas!"

                

                   Solução
                  

                    

                    

                    

                  

                

                static void saudar(String nome, int hora) {
    String cumprimento;

    if (hora >= 0 && hora <= 11) {
        cumprimento = "Bom dia";
    } else if (hora <= 17) {
        cumprimento = "Boa tarde";
    } else {
        cumprimento = "Boa noite";
    }

    System.out.println(cumprimento + ", " + nome + "!");
}

// Teste
saudar("Lucas",  9);  // Bom dia, Lucas!
saudar("Maria", 15);  // Boa tarde, Maria!
saudar("João",  21);  // Boa noite, João!
02
Calculadora com Retorno
Nível 1

Crie quatro métodos — somar, subtrair, multiplicar e dividir — que recebam dois double e retornem o resultado. Para divisão, verifique se o divisor é zero antes de dividir.

                

                   Solução
                  

                    

                    

                    

                  

                

                static double somar(double a, double b) {
    return a + b;
}

static double subtrair(double a, double b) {
    return a - b;
}

static double multiplicar(double a, double b) {
    return a * b;
}

static double dividir(double a, double b) {
    if (b == 0) {
        System.out.println("Erro: divisão por zero!");
        return 0;
    }
    return a / b;
}

// Teste
System.out.println(somar(10, 3));       // 13.0
System.out.println(dividir(10, 0));    // Erro: divisão por zero!
03
Boletim Escolar
Nível 2

Crie um sistema de boletim que combine múltiplos métodos. Implemente:

  • calcularMedia(double n1, double n2, double n3) — retorna a média
  • isAprovado(double media) — retorna true se média ≥ 6.0
  • exibirBoletim(String nome, double n1, double n2, double n3) — exibe tudo formatado
                

                   Solução
                  

                    

                    

                    

                  

                

                static double calcularMedia(double n1, double n2, double n3) {
    return (n1 + n2 + n3) / 3.0;
}

static boolean isAprovado(double media) {
    return media >= 6.0;
}

static void exibirBoletim(String nome,
        double n1, double n2, double n3) {

    double media = calcularMedia(n1, n2, n3);
    String situacao = isAprovado(media)
        ? "✅ Aprovado"
        : "❌ Reprovado";

    System.out.println("========================");
    System.out.println("Aluno: " + nome);
    System.out.println("Notas: " + n1 + ", " + n2 + ", " + n3);
    System.out.println("Média: " + media);
    System.out.println("Situação: " + situacao);
    System.out.println("========================");
}

// Teste
exibirBoletim("Lucas", 7.0, 8.5, 6.0);
exibirBoletim("Maria", 4.0, 5.0, 3.5);

Aluno: Lucas
Notas: 7.0, 8.5, 6.0
Média: 7.166...
Situação: ✅ Aprovado
========================
04
Sobrecarga — Área de Figuras
Nível 3

Crie um método calcularArea sobrecarregado para calcular a área de três figuras diferentes usando o mesmo nome:

  • Quadrado: recebe um double lado
  • Retângulo: recebe double largura, double altura
  • Círculo: recebe double raio — use Math.PI

Dica: como quadrado e círculo têm um parâmetro cada um, você vai precisar de tipos diferentes (int vs double) ou resolver de outra forma.

                

                   Solução
                  

                    

                    

                    

                  

                

                // Quadrado — recebe int (lado inteiro)
static double calcularArea(int lado) {
    return lado * lado;
}

// Retângulo — dois doubles
static double calcularArea(double largura, double altura) {
    return largura * altura;
}

// Círculo — double (raio decimal)
static double calcularArea(double raio) {
    return Math.PI * raio * raio;
}

// Teste
System.out.println(calcularArea(4));          // Quadrado: 16.0
System.out.println(calcularArea(5.0, 3.0));   // Retângulo: 15.0
System.out.println(calcularArea(2.0));       // Círculo: 12.566...

A diferença entre quadrado (int) e círculo (double) permite ao Java distinguir as duas sobrecargas de um parâmetro. calcularArea(4) chama a versão int; calcularArea(2.0) chama a versão double.

Resumo da Unidade

O que você aprendeu nesta unidade:

  • Um método é um bloco de código nomeado e reutilizável que executa uma tarefa específica.
  • Parâmetros são as variáveis declaradas na definição do método; argumentos são os valores passados na chamada.
  • O tipo de retorno define o que o método devolve: use void para nenhum retorno e o tipo desejado com return para devolver valores.
  • Sobrecarga permite criar vários métodos com o mesmo nome mas parâmetros diferentes — o Java escolhe qual chamar com base nos argumentos.
  • O escopo define onde uma variável existe: variáveis locais vivem só dentro do método; variáveis de bloco vivem só dentro do { } em que foram criadas.
  • Boas práticas: um método = uma responsabilidade, nomes descritivos, métodos curtos e evitar repetição de código (DRY).
